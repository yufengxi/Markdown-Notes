# 操作系统原理

## 一、操作系统引论

1. 单道批处理系统特点：
    1. 自动性
    2. 顺序性
    3. 单道性

2. 多道批处理系统：
    * 特征：
        1. 多道性
        2. 无序性
        3. 调度性
    * 优点：资源利用率高，系统吞吐量大
    * 缺点：无交互能力，平均周转时间长

3. 分时系统：
    * 定义：一台主机上连接了多个终端，并允许多个用户共享主机资源。
    * 特征：
        1. 多路性
        2. 交互性
        3. 及时性
        4. 独立性
    * 分时：按时间关系轮流使用资源
    * 多用户：同时使用计算机的用户>1
    * 单道分时系统
    * 具前台后台的分时系统
    * 多道分时系统

4. 实时系统：
    * 类型：
        1. 工业（武器）控制系统
        2. 信息查询处理
        3. 多媒体系统
        4. 嵌入式系统

* 操作系统定义：一组能有效组织和管理计算机硬件软件资源，和力对各类作业进行调度，以方便用户使用的程序的集合
* 操作系统目标：
    1. 有效管理计算机资源
    2. 方便用户使用计算机资源
    3. 可扩充性
    4. 开放性（可移植性）

* 操作系统基本特征：
    1. __并发__
        * 单CPU多道程序处理时，宏观上并发，微观上交替执行。
    2. 共享
    3. __虚拟__
        * 将一个物理实体被映射为若干个对应的逻辑实体
    4. 异步

* 操作系统主要功能：
    1. 处理器管理(CPU)
    2. 存储器管理(RAM)
    3. 设备管理(I/O)
    4. 文件管理(DISK)
    5. 用户接口

## 二、进程

### 2.1 前驱图和程序进行

* 程序顺序执行的特征：顺序性，封闭性（资源状态仅能由执行程序改变），可再现性（初始条件相同，则结构唯一）
* 程序并发执行的特征：间隔性，失去封闭性，不可再现性（因此要采取措施维持可再现性）

### 2.2 进程的描述

#### 2.2.1 进程性质

* 进程
    1. 定义：可并发执行的，具有独立功能的程序在一定数据集合上的一次执行过程（一次性+动态性），是操作系统进行资源分配和调度的基本单位
    2. 组成：程序+数据

* 进程的特征
    1. 动态性
    2. 并发性
    3. 独立性
    4. 异步性
    5. 结构特征（程序段，数据段，进程控制块（PCB））

* 进程的三种基本状态
    1. 运行：进程在CPU上执行
    2. 就绪：进程在内存中等待分配
    3. 阻塞：正在运行的进程收到阻塞不能执行，放弃处理器
    就绪和运行可互相转换，而运行到阻塞，阻塞到就绪是单向的过程
* 创建和终止状态：
    创建PCB -> 就绪 -> 运行 -> 终止

* 挂起操作：
    1. 引入原因：终端用户，负荷调节，操作系统的需要，父进程请求

#### 2.2.2 进程数据结构

* OS中管理控制的数据结构：**内存表，设备表，文件表，进程表**。PCB是进程存在的唯一标识

* 进程控制块（PCB块）中的信息：
    1. 进程标识符：
        * 外部标识符：创建时用户指定
        * 内部标识符：OS唯一指定
    2. 处理机状态：
        * 通用寄存器，指令寄存器，程序状态字，用户栈指针
    3. 进程调度：
        * 进程状态，优先级，阻塞原因
    4. 进程控制：
        * 程序和数据地址，资源清单等
* PCB组织方式：1. 线性表 2. 链接方式 3. 索引方式

### 2.3 进程控制

管理状态/用户状态

* 内核：OS中核心功能的集合
    1. 资源管理功能：进程管理，存储器管理，设备管理
    2. 支持功能： 中断处理（最基本功能），时钟管理，**原语操作**（不能中断，不可分割）

* 中断：程序执行过程中，终止CPU上现行程序运行（中断发生后，能够改变处理器内操作执行的顺序）

* 引起进程创建的事件：
    1. 操作系统初始化
    2. 用户登录（分时系统）
    3. 作业调度（外存-内存）
    4. 提供服务（输入/输出）
    5. 应用请求（进程自身）

* 进程的终止：
    从标识符叫所处PCB独处状态
* 进程的阻塞与唤醒：
    1. 向系统请求资源失败
    2. 等待某种操作完成
    3. 新数据尚未到来
    4. 等待新任务到达

### 2.4 线程

* 定义：操作系统进程中呢个狗独立执行的实体，进程的组成部分，是处理器调度的基本单位。进程中允许多个并发线程执行，但每一线程不独立占有进程资源。

### 2.5 进程同步

#### 2.5.1 基本概念

* 临界资源： 互斥资源，一段时间内只允许一个进程使用。

* 临界区：每个进程中访问临界资源的代码段。

* 同步机制遵循规律：空闲让进，忙则等待，有限等待，让权等待

#### 2.5.2 信号量机制

### 2.6 进程通信

1. 有界缓冲（低级通信）

2. 管道（pipe）：用于链接度进程和写进程实现他们通信的共享文件，单项信息传递信息方式，先进先出。
    1. 无名管道（存在于内核而非文件系统，无文件名）：

        ```c++
        Main()   {
            int filedes[2],i,pid;
            char inbuf[256];
        if (pipe(filedes)<0)         /* 创建可擦写的无名管道 */
            { perror("could not create unnamed pipe");exit(1);}
        if ((pid = fork())<0)       /* 创建子进程*/
            { perror("could not create subprocess"); exit(2); }
        if (pid>0)        			 /* 父进程*/
            { write(filedes[1],string,23); /* 向管道中写入数据 */
            printf("the parent process sended\n");
            wait(0);  }                 /*等待子进程结束*/
        if (pid==0)        	         /* 子进程*/
            { memset(inbuf, 0, sizeof(inbuf));
            read(filedes[0],inbuf,23);   /* 从管道接收数据 */
            printf("the subparent process received\n");
            printf("%s\n",inbuf); }
        exit(0);}
        ```

    2. 有名管道

3. 消息传递通信的方式

    1. 直接消息传递

    2. 间接通信方式（信箱）

## 三、处理器调度和死锁

### 3.1 处理器调度层次和算法

1. 高级调度（作业调度/长调度/接纳调度）
    决定外存上哪几个作业调入内存，并创建进程，分配资源，放入就绪队列
2. 低级调度（进程调度，短程调度）
    从就绪队列中选择哪个获得处理机，并将处理机分配给进程
3. 中级调度（内存调度，中级调度）
    为了提高内存率采取的外存补充内存的措施。

4. 处理器调度算法目标：
    1. 资源利用率
    2. 公平性
    3. 平衡性
    4. 策略强制执行
5. 实时系统目标：截止时间保证，可预测性

6. 分时系统目标：
    1. 响应时间
    2. 均衡性

7. 批处理目标：
    1. 周转时间短
    2. 系统吞吐量高
    3. 处理机利用率高

### 3.2 作业和作业调度

1. 作业：一组统一管理和操作的进程结婚，用户要求计算机系统完成的一项相对独立的工作

2. 作业控制块（JCB）：记录与作业有关的信息登记表，作业调度的唯一标识

3. 作业调度：
    * 系统的多道度：系统能够接纳作业的个数，由系统资源决定
    * 先来先服务(FCFS)算法
        * 计算平均周转时间和带权周转时间
    * 短作业（SJF）算法：到达的作业中选择运行时间最短的调用（平均周转时间最短）。能够提高系统**吞吐量**。
        > 可抢占
    * 高相应比优先调度算法（HRRF）
        $$相应比R_P = 1+ \frac{等待时间}{服务时间}$$
        每个时间段都运行相应比最高的作业
    * 平均周转时间 $FCFS >HRRF > SJF$
    * 优先级调度算法(PSA)
        > 可抢占

### 3.3 进程调度方式

* 抢占和非抢占
* 算法：
    1. 作业算法都可以通用
    2. 轮转(RR)
        时间片轮转算法是轮转中一种，如每个进程执行一定时间，之后进行下一个
    3. 多队列（MQ）
        按进程类型划分队列，个队列优先级逐渐降低
    4. 多级反馈队列(MFQ):多级队列+时间片
        1. 多级队列
        2. 时间片可变
        3. 进程不固定
    5. 基于公平原则的算法：
        1. 保证调度算法：每个进程享用CPU时间完全一致，对用户不公平
        2. 公平分享调度算法：按用户分配CPU时间

### 3.4 实时调度

1. 基本条件（必要信息）：就需时间，截止时间，处理时间，资源需求，优先级

2. 算法：
    1. 基于时钟中断的抢占算法
    2. 立即抢占调度算法
    3. 最早截止时间优先（EDF）算法（截止时间早的任务在队首，可用于非抢占和抢占调度）
    4. 最低松弛度优先算法（LLF）
    5. 有限集倒置

### 3.5 死锁

* 定义：一组进程中每个进程都在等待其他进程才能引发的事件。

* 产生原因：
    1. 竞争资源
    2. 进程推进非法

* 必要条件：
    1. 互斥
    2. 请求保持
    3. 不可抢占
    4. 环路等待

* 处理死锁方式：
    1. 预防死锁：
        破坏死锁的必要条件之一
        **层次分配法**
    2. 避免死锁
        最大需求矩阵，分配矩阵，需求矩阵
        **银行家算法**：先分配可分配的，待进程执行完能将其已有资源释放，之后进行下一进程
    3. 检测死锁
        画**资源分配图**，看是否有环路（有环路才会死锁）
    4. 解除死锁
        选择牺牲者，重新选择，饥饿
        **最小代价法**

## 四、存储器管理

### 4.1 存储器层次结构

* 主存：处理器能直接存取指令和指令存储器
* 外存：内存的补充，CPU不能直接存取指令数据，存放永久信息。

### 4.2 程序装入与链接

编辑，编译，链接，装入，运行

* 链接方法：
    1. 静态链接：运行前各目标模块先链接成装配模块，后不再拆开。
    2. 装入时动态链接
    3. 运行时动态链接
* 装入方式：
    1. 绝对装入方式
    2. 可重定位装入方式
    3. 动态运行时的装入方式

### 4.3 连续分配存储管理方式

* 描述分区的数据结构：空闲分区表，空闲分区链

* 基于顺序搜索的动态分区分配算法
    1. 首次适应算法（FF）
        > 尽可能利用低地址
    2. 循环首次适应算法（NF）
        > 查找从上次的空白区的下一个空白区
    3. 最佳适应算法(BF)
        > 按空白快大小递增排序，每次从最小空白区开始查
    4. 最坏适应算法(WF)
        > 按空白块大小递减排序，查找从最大空白区开始

* 基于索引搜索的动态分区分配算法
    1. 分类搜索
    2. 伙伴系统
        二分支（如申请256K，可将原512K分为两个256K伙伴分区分配）

* 紧凑技术：移动已分配信息，令空白区汇集能分配给进程，充分利用内存

### 4.4 对换

* 对换技术：将内存中暂不能运行的进程或暂时不用的程序或数据换到外存上，而将具备运行条件的换入内存。

1. 对换空间的管理
    * 文件区：离散分配（提高外存利用率）
    * 对换区：连续分配（兑换频繁，驻留时间短），位于磁盘中间

2. 换出的进程：
    * 先阻塞进程，后就绪进程
    * 低优先级；内存驻留时间；防止饥饿
3. 换入的进程：
    * 先就绪进程，后阻塞进程
    * 低优先级；外存的驻留时间

### 4.5 分页式存储管理方式

* 分页存储系统管理方式中地址结构：页号；偏移量
    > 逻辑地址$A = P * L + d$，页号$P=int[A\ / \ L]$， 页内地址$d=[A]\ mod\ l$
    > 物理地址 = 块号 * 块长 + 块内偏移（通过页表由逻辑地址映射到内存块物理地址）

* 页表

    * 产生：进程调入内存
    * 存放区：系统保护区
    * 数量和大小：每个进程一张，大小不一
    * 内容：页号，块号，存取控制字段

* 页面大小：一种机器大小固定

* 地址变换机构

* 快表：（页表一部分的副本）

* 页表级数：设物理块能存放N块，程序大小K页

## 五、虚拟存储器管理

### 5.1 概述

* 局部性原则：在较短时间内，程序执行仅限于某一部分，其存储空间也局限于某一区域

* 定义：指具有请求调入功能和置换功能，能够从逻辑上对内存容量上扩充的存储器系统。

* 特征：
    1. 离散性
    2. 多次性
    3. 对换性
    4. 虚拟性

### 4.2 请求分页存储方式

机制：页号 ， 物理块号， 状态位P，访问字段A，修改位M，外存地址
